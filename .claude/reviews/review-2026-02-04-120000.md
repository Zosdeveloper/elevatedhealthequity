## CRITICAL ISSUES (Must Fix)

### 1. ALL 13 PAGES: Forms never transmit data -- every submission is silently discarded
**Category**: Functionality / Data Loss
**Risk**: Visitors fill out forms, see a success message, but zero data reaches anyone. Every lead is lost.

All forms call `e.preventDefault()`, run client-side validation, then either `alert()` + `form.reset()` or show an in-page success div. No `fetch()`, no `<form action="...">`, no third-party integration.

Affected files and lines:
- `contact.html:467-501`
- `workshops.html:607-644`
- `about.html:622-660`
- `accelerator.html:646-683`
- `workshops/cultural-competency.html:741-778`
- `workshops/health-equity.html:741-778`
- `workshops/implicit-bias.html:741-778`
- All other 5 workshop subpages (same pattern)

**Fix**: Integrate a form backend (Formspree, Netlify Forms, AWS SES, etc.) and POST the data before showing success.

### 2. ALL 13 PAGES: Absolute-root navigation links (`/index.html`) break on non-root hosting and local file access
**Category**: Broken Links
**Risk**: If hosted in a subdirectory or opened via `file://` protocol, every nav link, footer link, and CTA returns 404. Approximately 200+ links site-wide.

Every page uses paths like `href="/index.html"`, `href="/workshops.html"`, `href="/contact.html"`.

**Fix**: Use relative paths (`index.html` from root, `../index.html` from `workshops/` subfolder). Or confirm deployment is always at domain root and document this requirement.

## HIGH PRIORITY (Strongly Recommended)

### 3. 11 PAGES: Missing `select` in CSS reset rule
**Files**: `index.html:21`, `workshops.html:21`, `about.html:21`, all 8 workshop subpages line 21
```css
/* Current -- missing select: */
button,input,textarea{font-family:inherit;font-size:inherit;border:none;outline:none}

/* contact.html:21 and accelerator.html:21 correctly have: */
button,input,textarea,select{font-family:inherit;font-size:inherit;border:none;outline:none}
```
**Impact**: `<select>` elements on 11 pages do not inherit the Montserrat font and retain browser-default borders/outlines, creating visual inconsistency.
**Fix**: Add `select` to the reset rule on all affected pages.

### 4. ALL PAGES: Form validation provides no accessible error feedback
**Files**: All pages with forms
**Impact**: Errors are indicated solely by changing `borderColor` to red. Screen reader users get zero feedback. WCAG 3.3.1 (Error Identification) failure.
- `contact.html`, `workshops.html`, and all 8 workshop subpages: only red border, no text message
- `about.html`: has `.error-msg` spans in HTML but the JS uses `style.borderColor` instead of toggling them
- `accelerator.html`: only page that correctly shows `.error-msg` text and uses `.error` CSS class
**Fix**: Add `aria-invalid="true"` to invalid fields, show visible text error messages, use `aria-describedby` to link them.

### 5. ALL PAGES: Links are visually indistinguishable from body text
**File**: All pages, line 19: `a{color:inherit;text-decoration:none}`
**Impact**: WCAG 1.4.1 failure. Any non-button, non-nav link in body content is invisible to sighted users.
**Fix**: Scope the reset to navigation/button contexts only, or add underline/color distinction for body links.

### 6. ALL PAGES: No `prefers-reduced-motion` support
**Files**: `index.html:63-71` (scroll reveal animations), `index.html:249` (partner carousel `animation:scroll 25s linear infinite`), all pages with `.reveal` transitions
**Impact**: Users with vestibular disorders experience discomfort from automatic animations. WCAG 2.3.3.
**Fix**: Add `@media(prefers-reduced-motion:reduce){.reveal,.reveal-left,.reveal-right{opacity:1;transform:none;transition:none}.partners-track{animation:none}}`.

### 7. ALL PAGES: No favicon, no Open Graph meta, no canonical URL
**Impact**: Browsers request `/favicon.ico` and 404. Social sharing shows no preview. Search engines may index duplicate URLs.
**Fix**: Add `<link rel="icon" ...>`, `<meta property="og:title/description/image">`, and `<link rel="canonical" ...>` to all pages.

### 8. ALL PAGES: Duplicate `.btn svg` CSS rule
**File**: All pages, lines ~90 and ~92
```css
.btn svg{width:1rem;height:1rem;flex-shrink:0}  /* line 90 */
.btn:hover svg{transform:translateX(4px)}         /* line 91 */
.btn svg{transition:transform .3s}                /* line 92 -- DUPLICATE selector */
```
**Impact**: Minor. The second `.btn svg` block should be merged with the first.
**Fix**: Merge into `.btn svg{width:1rem;height:1rem;flex-shrink:0;transition:transform .3s}`.

### 9. ALL FORMS: No spam/bot protection
**Impact**: When a backend is connected, open forms will be immediately targeted by bots.
**Fix**: Add honeypot field, CAPTCHA, or rate limiting.
